# UI – Common Badge

이 문서는 서비스 전반에서 사용되는 **Badge UI (상태/라벨 표시)**의 공통 규칙과 UX 정책을 정의한다.

본 문서는 `ui/index.md`, `ui/common/button.md`, `ui/common/empty-state.md`의 원칙을 따른다.

---

## 1. 역할 정의 (Role)

Badge는 **객체의 상태, 속성, 분류 정보를 짧고 명확하게 표시**하기 위한 보조 UI 요소이다.

### Badge의 목적

- 상태를 빠르게 인지하도록 돕는다
- 리스트/카드 내 정보 스캔 효율을 높인다
- 시각적 강조를 최소 비용으로 제공한다

---

## 2. 사용이 적합한 경우

- 주문 상태 (결제완료 / 배송중 / 완료)
- 재고 상태 (품절 / 한정 수량)
- 상품 속성 (NEW / BEST)
- 알림/이벤트 상태

### 사용이 부적합한 경우

- 긴 설명이 필요한 정보
- 사용자 액션을 요구하는 경우 → Button 사용

---

## 3. CVA 기반 Variant 설계

Badge는 **CVA(Class Variance Authority)** 기반으로 variant를 관리한다.

### 3.1 Intent Variant (의미)

- `default`: 일반 정보
- `success`: 긍정적 상태 (완료, 정상)
- `warning`: 주의 상태 (한정, 임박)
- `error`: 부정적 상태 (품절, 실패)
- `info`: 중립 정보

### 3.2 Size Variant

- `sm` (default)
- `md`

> Badge는 시각적 보조 요소이므로 `lg`는 사용하지 않는다.

---

## 4. 텍스트 규칙

- Badge 텍스트는 **짧고 명확**해야 한다.
- 1~2단어를 권장한다.
- 줄바꿈은 허용하지 않는다.

예시:

- ⭕ 배송중
- ⭕ 결제완료
- ❌ 결제가 완료되었습니다

---

## 5. 색상 사용 원칙

- 색상은 의미 전달을 보조하는 역할만 한다.
- 색상 단독으로 상태를 전달하지 않는다.
- 텍스트로 의미가 반드시 전달되어야 한다.

---

## 6. 배치 가이드

- 카드, 리스트, 테이블 셀 내부에 배치한다.
- 제목 옆 또는 우측 상단 배치를 권장한다.
- 여러 Badge를 동시에 사용할 경우 간격을 확보한다.

---

## 7. 접근성 (Accessibility)

- Badge 텍스트는 스크린 리더에 그대로 전달된다.
- 색상 대비는 WCAG 기준을 충족해야 한다.

---

## 8. Mobile UX 고려

- 작은 화면에서도 읽기 쉬운 크기를 유지한다.
- 터치 대상이 아니므로 과도한 크기 확대는 지양한다.

---

## 9. Button / Link와의 관계

- Badge는 클릭 가능한 요소로 사용하지 않는다.
- 액션이 필요한 경우 Button 또는 Link를 사용한다.

---

## 10. 확장 고려

- 아이콘 포함 Badge
- Status 코드 기반 Badge 매핑
- 다크모드 대응

---

### 요약

- Badge는 상태를 빠르게 전달하는 보조 UI
- CVA 기반 intent / size 관리
- 짧고 명확한 텍스트 사용
- 액션 요소로 사용하지 않는다
